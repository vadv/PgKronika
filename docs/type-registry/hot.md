# Классы 2, 3 и 10: HOT-данные

HOT-данные лежат в хвосте сегмента или используются вместе с ним:

- события для таймлайна;
- словари;
- предрассчитанные графики.

## Класс 2: события

| `type_id` | Содержимое |
|-----------|------------|
| `2_001_001` | секция `events`, раскладка описана в [`segment-format.md`](../segment-format.md) |

`event_type` — отдельный `u32` внутри секции `events`. Он ведется отдельной
таблицей в коде, потому что это не секция PGM и не требует отдельного
`type_id` на каждое событие.

Стартовый набор `event_type`:

| Событие | Смысл |
|---------|-------|
| `pg_restart` | рестарт PostgreSQL |
| `stats_reset` | reset статистики |
| `checkpoint_req` | checkpoint по требованию или слишком частые checkpoint |
| `autovacuum_start` | начало autovacuum |
| `autovacuum_end` | завершение autovacuum |
| `lock_wait_begin` | начало ожидания lock |
| `lock_wait_end` | завершение ожидания lock |
| `oom_kill` | OOM kill по данным cgroup или логов |
| `replication_lag_high` | высокий replication lag |
| `collector_gap` | пропуск сбора |
| `segment_early_seal` | досрочное запечатывание сегмента |
| `source_unavailable` | источник временно недоступен |
| `permission_denied` | источник доступен не полностью из-за прав |
| `log_gap` | читатель логов обнаружил возможную потерю строк |

События должны быть тонкими: метка на таймлайне плюс ссылки на сущности
(`pid`, `str_id`, типизированная секция). Богатая числовая информация остаётся
в секциях снимков. В текущей области логов это `1_022_001`, `1_024_001`,
`1_026_001`, `1_028_001` и `1_029_001`; autovacuum, lock waits, temp files и
csvlog остаются отдельными отложенными расширениями.

Если тонкое событие ссылается на строку типизированной секции, связь задается
соглашением: `event_type` однозначно определяет целевой `type_id`, адресация
идёт по равенству `ts` события и строки секции, а `ref_str` содержит ключ
сущности (`pattern` для ошибок, `table_name` для autovacuum и так далее). Для
синглтон-строк `ref_str = 0`.

Числовую полезную нагрузку запрещено упаковывать в JSON-строки словаря без
сильной причины. Уникальные числа делают каждую JSON-строку уникальной и
раздувают `dict.strings`.

| `type_id` | Содержимое |
|-----------|------------|

## Класс 10: графики

`agg`:

- `0` — `gauge`, рисуется значение;
- `1` — `counter_rate`, рисуется дельта за секунду.

`values_are` не хранится в секции. Это атрибут документации реестра: он
фиксирует формулу, по которой код записи вычислил точки. Соответствие должно
выполняться: `rate` требует `agg = 1`, `gauge` и `derived` требуют `agg = 0`.
Формула расчёта — часть версии графика. Смешивать серии с разными
единицами измерения в одной секции графика запрещено; для них используются разные
`type_id`.

| `type_id` | График | `entity` | Серии | Единица | `agg` | `values_are` | Источник |
|-----------|--------|----------|-------|---------|-------|--------------|----------|
| `10_001_001` | checkpoints | `0` | `timed`, `requested` | `ops/s` | `1` | `rate` | `1_006_001` |
| `10_002_001` | TPS | `0` | `commit`, `rollback` | `tx/s` | `1` | `rate` | `1_005_001` |
| `10_003_001` | blocks | `0` | `read`, `hit` | `blk/s` | `1` | `rate` | `1_005_001` |
| `10_004_001` | tuples | `0` | `ins`, `upd`, `del` | `tup/s` | `1` | `rate` | `1_005_001` |
| `10_005_001` | WAL bytes | `0` | `bytes` | `B/s` | `1` | `rate` | `1_007_001`, `1_007_002` |
| `10_006_001` | backends by state | `0` | `active`, `idle_in_xact`, `waiting` | `n` | `0` | `gauge` | `1_001_001` |
| `10_007_001` | host CPU | `0` | `user`, `system`, `iowait`, `steal` | `%` | `0` | `derived` | `1_102_001` |
| `10_008_001` | host memory | `0` | `used`, `cached`, `available` | `B` | `0` | `gauge` | `1_104_001` |
| `10_009_001` | PSI | `0` | `cpu_some`, `mem_full`, `io_full` | `%` | `0` | `gauge` | `1_107_001` |
| `10_010_001` | disk IOPS, data dir | `device` | `read`, `write` | `io/s` | `1` | `rate` | `1_108_001` + `1_112_001` |
| `10_011_001` | disk throughput | `0` | `read`, `write` | `B/s` | `1` | `rate` | `1_108_001` |
| `10_012_001` | disk latency | `0` | `read`, `write` | `ms` | `0` | `derived` | `1_108_001` |
| `10_013_001` | network | `0` | `rx`, `tx` | `B/s` | `1` | `rate` | `1_109_001` |
| `10_014_001` | TCP retransmit | `0` | `retrans`, `syn_retrans` | `seg/s` | `1` | `rate` | `1_110_001` + `1_111_001` |
| `10_015_001` | locks waiting | `0` | `waiting_backends` | `n` | `0` | `gauge` | `1_011_001`, `1_011_002` |
| `10_016_001` | autovacuum | `0` | `running` | `n` | `0` | `gauge` | `1_012_001` |
| `10_017_001` | replication lag | standby: `0`; primary: `application_name/client_addr` | `replay_lag` | `s` | `0` | `gauge` | `1_015_001` + `1_016_001` |
| `10_018_001` | wraparound headroom | `0` | `max_age` | `xid` | `0` | `gauge` | `1_018_001` |
| `10_019_001` | WAL records | `0` | `records`, `fpi` | `ops/s` | `1` | `rate` | `1_007_001`, `1_007_002` |

Для `10_010_001` пишется отдельный экземпляр на устройство, `entity = device`.
Для `10_016_001` серия `running` равна числу строк `1_012_001` с
`is_autovacuum = true`; отсутствие секции `1_012_001` даёт `0`.
Для `10_017_001` на standby пишется один экземпляр `entity = 0` с
`replay_lag` из `1_015_001`; на primary пишется отдельный экземпляр на реплику
из `1_016_001`, `entity = application_name/client_addr`. `1_015_001` не
заменяет детализацию по репликам и не несёт retained WAL по слотам.
Для wraparound-графиков XID-возраст и MXID-возраст не смешиваются в одну шкалу.
`frozen_xid_age` агрегируется как `max(frozen_xid_age)`, `min_mxid_age` — как
`max(min_mxid_age)`, оба по строкам реальных баз; `NULL` shared-строки
игнорируется. `10_018_001` — XID-график; MXID требует отдельного `type_id`.
Headroom считается отдельно: для XID
`autovacuum_freeze_max_age - frozen_xid_age`, для MXID
`autovacuum_multixact_freeze_max_age - min_mxid_age`.

Производные серии, например `latency = time_delta / ops_delta`, считаются при
запечатывании сегмента. В секции графика они уже лежат как готовые точки и
рисуются как `gauge`, поэтому `agg = 0`. Формула — часть кода графика и
его версии: изменение формулы требует нового `type_id`.
