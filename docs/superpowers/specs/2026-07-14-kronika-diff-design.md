# Контракт `kronika-diff`

`kronika-diff` — домен-независимый интерпретатор контрактов реестра. Знание о
версиях PostgreSQL, GUC и составе системных views остаётся в
`kronika-registry`.

## Вычисление

Для соседних реальных точек одной серии:

- `delta = current - previous`;
- целые значения вычитаются в `i128`;
- `rate = delta / dt`, где `dt` — фактический интервал;
- отрицательная delta даёт `NoData::Reset`;
- `dt <= 0` даёт `NoData::Anomaly`;
- экстраполяция и компенсация wraparound не выполняются.

Классы берутся только из контракта: `Cumulative` идёт в counter-путь, а
`Gauge`, `Label` и `Timestamp` не дифференцируются. Поэтому min/max/mean/stddev,
размеченные в реестре как gauges, остаются исходными значениями.

## Идентичность серии

`TypeContract.identity` перечисляет Label-колонки сущности. Линтер проверяет
существование и класс каждой колонки. Если `identity` пуст, используется
`sort_key` без `ts`; пустой результат означает singleton-серию.

Для `pg_stat_statements` ключ равен `(queryid, userid, dbid, toplevel)`.
Nullable `queryid` остаётся `Null` в ключе: при выключенном `compute_query_id`
строки группируются по остальным полям, без выдуманного идентификатора.

## Пропуски и сбросы

Каждая нечисловая точка имеет явную причину:

- `FirstPoint` — нет предшественника;
- `Reset` — счётчик уменьшился;
- `Gap` — между точками нет покрытия;
- `Anomaly` — время не возрастает;
- `NotCollected` — контракт сообщает, что значение не собиралось.

Ни одна причина не преобразуется в измеренный ноль. Через coverage gap серия не
протягивается. Для `SnapshotFull` исчезновение завершает серию; для `Changed`
отсутствие строки при непрерывном покрытии означает отсутствие изменения.

## Границы ответственности

Reader строит logical section, группирует строки по identity и формирует
типизированные series. `kronika-diff` вычисляет только значения пары.
HTTP-слой сериализует результат и применяет лимиты; он не повторяет декодирование
и не содержит PostgreSQL-специфичных исключений.

## Принятые ограничения

- Дифф вычисляется при чтении и не сохраняется в segment.
- `delta < 0` — единственный общий сигнал reset; источник reset не угадывается.
- Переиспользование OID между соседними snapshot может соединить разные
  сущности, если новое значение не меньше старого. Для точного решения нужен
  объявленный реестром epoch, которого нет у всех секций.
- GUC-зависимые интервалы становятся `NotCollected`, если гейт выключен или
  неизвестен в наблюдавшихся состояниях. Между снапшотами переключение не видно;
  session-level GUC считается одинаковым у collector и contributing backends.
