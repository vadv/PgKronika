# Спека: kronika-diff — производные над кумулятивными счётчиками

Крейт: `crates/kronika-diff` (сейчас стаб на 5 строк). Стоит поверх query-слоя
`kronika-reader`. Ветку заводим при реализации.

## Цель

`pg_kronika-web` отдаёт сырые накопленные счётчики pg_stat_*. Из них не видно ни
запросов в секунду, ни cache hit — только монотонно растущие числа. kronika-diff
превращает их в приросты (`delta`) и скорости (`rate`) за интервал. Слой блокирует
charts, UI и детекцию аномалий.

## Принцип

Дифф — домен-агностичный интерпретатор. Он не знает имён полей PostgreSQL,
версий, GUC. Всё, что ему нужно про секцию, уже выражено в контракте реестра:
класс колонки, семантика сбора, ключ идентичности. Правило вычисления одно и
простое.

## Ядро

Дифф считается на паре соседних реальных снапшотов, отдельно на каждую серию (по
ключу идентичности), затем агрегируется.

- `delta = v2 − v1`. Для `I64`/`U64` вычитание в `i128`; тайминги `F64` — как есть.
- `delta < 0` → счётчик сбросили или провернули → точка не выдаёт число, статус
  `NoData { Reset }`. Без компенсации `+prev`, без реконструкции.
- `rate = delta / dt`, где `dt` — реальный интервал между снапшотами, результат
  `f64`.
- Экстраполяции нет: реальные точки, реальный `dt`. Дробных дельт на целых
  счётчиках не возникает by construction (в отличие от Prometheus).

## Решения

Приняты владельцем (2026-07-14):

1. **Query-time.** Считаем на чтении. При записи снапшота ничего не
   материализуем — семантику меняем без перезаписи сегментов.
2. **`delta < 0` → `Reset` → пропуск точки.** Отрицательный прирост и есть детект
   сброса — осознанное упрощение шага 1 по решению владельца. Per-domain
   `reset_source` не нужен, `delta < 0` его заменяет. Сигнальные декларации для
   честности (`gated_by`) и точности OID (`entry_epoch`) вынесены в § «Будущие
   декларации»: они дополняют правило, а не отменяют его.
3. **Битая точка — статус, не число.** `NoData { reason }`: не ноль (маскирует
   потерю), не минус (мусор), не пропуск строки.
4. **`resets()` не вводим.**
5. **Агрегат не скрывает деградацию.** Рядом с агрегатом по валидным сериям —
   `series_total` и `series_excluded { reason: count }`.

## Контракт: одно расширение

Дифф группирует строки в серии по идентичности сущности во времени. `sort_key`
для этого не годится: это физический порядок (у pg_stat_statements — `dbid,
userid, ts`, без `queryid`). Добавляем одно поле:

- `TypeContract.identity: &'static [&'static str]` — Label-колонки, задающие
  сущность серии: statements → `queryid, userid, dbid, toplevel`; database →
  `datid`; user_tables → `datid, relid`; user_indexes → `datid, indexrelid`.
  Линтер: каждое имя — колонка класса `Label`.

Пустой `identity` (секция не размечена) значит: ключ серии берётся из `sort_key`
без `ts`. Для секций с `sort_key = (сущность…, ts)` это и есть сущность (os_cpu →
cpu_id, pg_stat_io → backend_type/object/context), поэтому многострочные секции
не схлопываются в одну серию. Для синглтона (`sort_key = ts`) ключ пуст — одна
серия на секцию, что для него верно. `identity` объявляется явно только там, где
`sort_key` недостаточен: pg_stat_statements сортируется по dbid/userid, но сущность
добавляет queryid/toplevel. Gauge-только секции (activity, locks) в дифф не входят —
у них нет кумулятивов. Nullable-колонка в ключе (`queryid` при `compute_query_id =
off`) схлопывает NULL-строки в одну серию по остатку ключа; свёртка (шаг 3) обязана
это учесть — связано с `NotCollected` ниже.

Остальное (`ColumnClass`, `semantics`, `columns`) уже в контракте.

## Классы колонок

Из реестра, уже есть (`ColumnClass`):

- `Cumulative` → counter-путь (delta/rate).
- `Gauge`, `Label`, `Timestamp` → as-is.

Закрывает класс ошибок «rate по gauge»: тип это свойство данных, не суффикс имени.
Prometheus и VictoriaMetrics тип не знают и молча считают rate по gauge.

## Разреженные серии

Два разных «нет данных», толкуются по `semantics` контракта:

- **Coverage-gap** — окно не покрыто сегментами (reader отдаёт `Gap`): снапшотов
  нет. Пара через coverage-gap → `NoData { Gap }`; первая точка после →
  `FirstPoint`. Мостов нет ни для какой семантики.
- **Семантическая разреженность** — окно покрыто, строки серии нет в части
  снапшотов:
  - `Changed` (только изменившиеся строки плюс baseline): отсутствие строки =
    «счётчики не менялись»; мост к последнему виденному значению легитимен, delta
    накопленная, dt реальный.
  - `SnapshotFull`: отсутствие строки = сущность исчезла; серия завершена;
    повторное появление = новая серия, `FirstPoint`.

## Публичный API

Точка диффа:

- `Value { delta, rate, dt }`.
- `NoData { reason }`, `reason ∈ { Reset, Gap, FirstPoint, Anomaly, NotCollected }`:
  - `Reset` — `delta < 0`.
  - `Gap` — пара пересекает coverage-gap.
  - `FirstPoint` — предшественника нет: начало серии, первая точка после Gap, или
    предыдущий сегмент удалён ретеншном.
  - `Anomaly` — `dt <= 0`: время между снапшотами не растёт, данные битые.
  - `NotCollected` — источник колонки в окне отключён: GUC `track_io_timing` off →
    `blk_*_time` нули не от быстрого IO; `compute_query_id` off → `queryid`
    невалиден как ключ. Отдаём статус, не `Value { delta: 0 }` — иначе выключенный
    тайминг читается как «нулевая задержка». Опирается на декларацию `gated_by`
    (§ «Будущие декларации»); в шаге 1 не реализовано.

Ядро — чистая функция:

`diff_pair(prev, cur, prev_ts, cur_ts) -> DiffPoint`

Детерминирована, без I/O: два `Scalar` (`Int`/`Float`) и их таймстампы — без
`ColumnClass`, чтобы ядро не знало домена; маршрутизацию Cumulative/Gauge делает
свёрточный слой. Query-слой группирует строки в серии по ключу и внутри серии
сворачивает соседние по времени пары. Выдача `section()` отсортирована по
`sort_key` (`dbid, userid, ts`), не по ключу серии, поэтому строки одной серии в
ней НЕ соседние — группировка по кортежу ключа (в реализации `BTreeMap` для
детерминизма выдачи; `FxHash`/`ahash` O(rows) — перф-оптимизация позже), а не
пересортировка всей выдачи O(rows·log rows). reach-back к последнему
снапшоту перед окном — один на страницу, не на серию.

Эскиз строки ответа /v1: identity-колонки как есть, `Gauge`-колонки as-is, для
каждой `Cumulative`-колонки `{ delta, rate } | { nodata: reason }`; агрегатные
ответы дополняются `series_total` / `series_excluded`.

## reset_metadata: что берём, что нет

Секция `1_020_001` reset_metadata несёт per-domain reset-таймстампы, GUC
(`compute_query_id`, `track_io_timing`, `track_wal_io_timing`) и версии
расширений.

- **Детект сброса на неё НЕ завязан.** Сброс ловится через `delta < 0`; per-domain
  reset-таймстампы для этого не нужны.
- **GUC-поля идут в `NotCollected`.** `track_io_timing`/`compute_query_id` в
  состоянии off делают соответствующие колонки невалидными — это честность, не
  детект сброса (см. `NotCollected` и § «Будущие декларации»).

## OID reuse

`SnapshotFull`-правило «исчезла из снапшота = серия завершена» не закрывает
DROP+CREATE между двумя соседними снапшотами: переиспользованный
`relid`/`indexrelid` сваривает смерть таблицы A и рождение таблицы B в одну серию
с фиктивной положительной дельтой. `entry_epoch` (`stats_since`) для
tables/indexes недоступен — таких колонок в контрактах нет.

Решение шага 1: риск ПРИНЯТ. Частота DROP+CREATE с реюзом OID между двумя
соседними снапшотами низка; падение поймается как `Reset` (`delta < 0`), но
рост-без-падения пройдёт как валидная дельта. Точное закрытие — прокси-эпоха
`relfilenode` (меняется при переписывании таблицы) как `entry_epoch` в снапшотах
tables/indexes — вынесено в § «Будущие декларации».

## Будущие декларации контракта

Не в этом MR. Закрывают Б1 (честность) и Б2 (OID reuse); реализуются с ядром
диффа (шаги 2-4), не в контрактном шаге 1:

- `gated_by` — какой GUC-флаг reset_metadata обнуляет достоверность колонки
  (`blk_*_time` → `track_io_timing`; `queryid` → `compute_query_id`). Off →
  `NotCollected`.
- `entry_epoch` — Timestamp-колонка секции, чья новизна = «запись пересоздана»
  (statements → `stats_since`; tables/indexes → `relfilenode` как прокси, надо
  добавить в снапшот).

Тип: строковый слайс, как у `identity`, для этих деклараций не годится —
`gated_by` ссылается на поле ДРУГОЙ секции (reset_metadata), `entry_epoch` — на
одиночную колонку своей. Вводить типизированные ссылки (`ColumnRef` /
`SectionColumnRef`), не расширять строковый паттерн.

## Правила

- **R1.** `Cumulative` — counter-путь; `Gauge`/`Label`/`Timestamp` — as-is.
- **R2.** `delta < 0` → `Reset` (пропуск). Единственный детект сброса.
- **R3.** Без экстраполяции: реальные точки, реальный dt.
- **R4.** Через coverage-gap не дотягиваться; семантический мост `Changed`-серий
  при покрытом окне легитимен. Первая точка серии и первая после gap →
  `FirstPoint`.
- **R5.** Целые дифы до float: `i128`-вычитание, результат в `f64`. `F64`-колонки
  — как есть.
- **R6.** Диф per-series, потом агрегация; агрегат несёт счётчики деградации.
- **R7.** min/max-поля (класс задаётся контрактом) не дифференцировать.

## Тесты

Ядро `diff_pair` — unit-тесты без моков:

- Монотонный рост → `Value`, delta точная.
- `U64` на входе → точная `i128`-дельта, дельта не дробна.
- `v2 < v1` → `Reset` (без разбора причин).
- `dt <= 0` → `Anomaly`.
- `Gauge`-колонка counter-путём не идёт.
- Первая точка серии → `FirstPoint`.

Свёртка серий (query-слой):

- `Changed`-серия: строка отсутствует в средних снапшотах покрытого окна → мост,
  delta накопленная.
- `SnapshotFull`-серия: исчезла и появилась → старая завершена, новая с
  `FirstPoint`.
- Агрегация: `Reset` в одной серии не искажает сумму остальных; ответ несёт
  `series_excluded` с причиной.
- Инварианты: delta счётчика не отрицательна и не дробна на целых; rate конечен.

Линтер контракта: `identity` ссылается только на колонки класса `Label`.

BDD: `pg_stat_reset` между двумя сборками → серии секции в окне получают `Reset`
(значения упали), соседние точки валидны.

шаг 4 отложен.

4. Типизированные декларации `gated_by`/`entry_epoch`, `NotCollected` и

5. /v1-эндпоинт диффа. ✔ BDD-сценарий на сам дифф — отложен (существующие 79
   сценариев зелёные).

## Осознанно отложено

- Материализация дифов при seal — выбран query-time.
- Порог частичного сброса (1/8 у VictoriaMetrics) — `delta < 0` это просто
  `Reset`.
- `resets()`, экстраполяция, коррекция wraparound (счётчики PG — bigint).
- Per-domain reset-детект через reset_metadata — `delta < 0` его заменяет.
- `gated_by`/`entry_epoch` (`NotCollected`, OID-эпоха) — § «Будущие декларации»,
  с ядром диффа, не в контрактном MR.
