# OS Wave 1: procfs core metrics

Date: 2026-07-04.

Wave 1 adds six host-wide procfs sections to PgKronika:

- `1_102_001` `/proc/stat` CPU ticks
- `1_103_001` `/proc/stat` misc counters
- `1_104_001` `/proc/meminfo`
- `1_105_001` `/proc/loadavg`
- `1_106_001` `/proc/vmstat`
- `1_107_001` `/proc/pressure/{cpu,memory,io}`

The registry contract for these sections is in `docs/type-registry/os.md`.

## Scope

Every Wave 1 row carries `scope: u8`. These procfs files expose host-wide kernel
state even when the collector runs inside a container, so Wave 1 writes
`scope=0` (`host`) for all six sections. Later OS waves can use the remaining
scope values for pod network namespace, process namespace, and cgroup data.

## Collection

The collector reads Wave 1 files through `kronika_source_os::ProcFs`. The root is
`/proc` by default and can be overridden with `KRONIKA_PROC_ROOT` for tests or
host-mounted deployments. Each read is bounded by `MAX_PROC_FILE_BYTES`.

`SourceKind::OsCore` gates the six sections as one source. The interval is
controlled by `KRONIKA_OS_CORE_INTERVAL_S`, defaulting to 10 seconds. The
collector reads procfs synchronously before creating `SectionBuffers` and the
segment dictionary.

Read or parse failure skips only the affected section and emits
`collection_degraded`; the collector does not fabricate zero rows or zero field
values. Nullable fields represent values that the running kernel did not expose.

## rpglot floor

Wave 1 covers the rpglot fields for the same procfs files:

- CPU: aggregate and per-CPU ticks, including `guest` and `guest_nice`
- stat misc: `ctxt`, `processes`, `procs_running`, `procs_blocked`, `btime`
- meminfo: total/free/available memory, buffers, cache, slab reclaimability,
  swap, dirty, and writeback fields
- loadavg: 1/5/15 minute averages plus running and total process counts
- vmstat: paging, swap, reclaim, scan, major fault, and OOM kill counters
- PSI: `some` and `full` averages/totals for CPU, memory, and IO

PgKronika differs where the source can be absent or ambiguous: missing vmstat and
meminfo keys are `NULL`, and CPU PSI `full_*` fields are `NULL` because the
kernel does not expose a CPU `full` line.

## Tests

Parser and codec tests cover each section directly. BDD uses a fixture procfs
root through `KRONIKA_PROC_ROOT`, so end-to-end assertions are independent of the
host running the test.

## Deferred

Wave 1 does not collect process, disk, network, mountinfo, or cgroup metrics.
Those need separate contracts for source scope, identity, cardinality limits,
privacy, and degradation.
