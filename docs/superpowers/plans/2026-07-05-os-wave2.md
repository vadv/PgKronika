# OS Wave 2 (disk / network / mountinfo / topology) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the Wave 2 OS metric sections — `/proc/diskstats`, `/proc/net/{dev,snmp,netstat}`, mountinfo-based disk attribution, and CPU topology — to the collector, at parity with (or above) the reftool reference, on the Wave 1 procfs foundation.

**Architecture:** Each `/proc` file gets a pure parser `fn parse(content: &str) -> Result<Vec<Row>, ParseError>` in `crates/kronika-source-os/src/proc/` (host-independent, unit-tested on strings), separate from file reading via `ProcFs`. Each section gets a `#[derive(Section)]` codec in `crates/kronika-registry/src/codec/`. Counter sections (diskstats, net×3) extend the existing synchronous `OsCore` collection path (`snapshot_full` every tick). The mountinfo attribution engine lives in a dedicated `crates/kronika-source-os/src/mount.rs`. Mountinfo and topology are `on_change` sections emitted once per segment via the existing `SEGMENT_OPEN_SOURCES` re-arm (same pattern as `pg_settings`).

**Tech Stack:** Rust (edition 2024, MSRV 1.96), `#[derive(Section)]` registry codec, `ProcFs`/`KRONIKA_PROC_ROOT` fixtures, cucumber BDD, statvfs via `rustix::fs::statvfs` behind the fs abstraction.

## Global Constraints

- **reftool-floor (HARD gate):** every section must collect **no fewer** fields than reftool gathers from the same file. More is allowed; fewer is a blocker. Verified floors are inlined per task below. reftool source: `/home/vadv/Projects/reftool/crates/reftool-core/src/collector/procfs/parser/{disk,network}.rs` + `storage/model/system.rs`.
- **Never fabricate zeros:** on read/parse error, skip the section and log `collection_degraded` (per-section). Only `instance_metadata` reads are hard errors; OS sections degrade.
- **Raw units:** store kernel-native units (sectors of 512 B, ms, bytes, packets). Reader converts. Do **not** multiply sectors→bytes in the collector.
- **scope column** (`u8`, `#[column(l)]`) on every OS row: `host=0, pod=1, pod_net=2, container=3, unknown=4` (`OsScope::as_u8`). diskstats and mountinfo = `host` (device-level, describes the node's device). net×3 = `pod_net` when a container is detected, else `host` (these files are network-namespace scoped — the first place scope genuinely differentiates). topology = `host`.
- **type_id numbers are internal constants** — the exact `1_108…1_113` values are fixed by `docs/type-registry/os.md`; do not renumber. Reader maps id→name.
- **Language:** code / rustdoc / BDD in English; commit messages in Russian; no `Co-Authored-By`.
- **Gate (must pass before每 commit is considered done):** `export PATH="$HOME/.cargo/bin:$PATH"` then `cargo fmt --all -- --check && cargo clippy --workspace --all-targets -- -D warnings && cargo test --workspace && cargo run -p xtask -- check-deps`. clippy: derive `Eq` when no `f64` fields, else `PartialEq`.
- **!Send boundary:** `SectionBuffers`/`Interner` must not be held across `.await`. Wave 2 collection is synchronous (no await) — keep it that way.

**Interfaces:** Produces `pub struct FsSpace { pub total_bytes: i64, pub free_bytes: i64 }` and, on `ProcFs` (or a sibling), `pub fn statvfs(&self, mount_point: &str) -> Option<FsSpace>`. ## Task 8: scope refinement for network + detect_container consistency
