# OS Wave 2: disk, network, mountinfo, topology

This document records the Wave 2 OS metric contract and implementation shape.
It is a technical reference for the branch, not an execution checklist.

## Goal

Add the next OS metric sections on top of the Wave 1 procfs foundation:

- `/proc/diskstats`
- `/proc/net/dev`
- `/proc/net/snmp`
- `/proc/net/netstat`
- `/proc/self/mountinfo` with filesystem capacity
- `/proc/cpuinfo` topology plus sysfs max CPU frequency

The collector keeps kernel-native units: 512-byte sectors, milliseconds, bytes,
packets, and raw counter values. Readers perform any presentation conversion.
On read or parse failure, an OS section degrades independently and logs
`collection_degraded`; missing data is not replaced with zero.

## Scope Contract

Every Wave 2 OS row carries `scope u8`:

| Scope | Value | Wave 2 use |
|-------|-------|------------|
| `host` | `0` | `diskstats`, `mountinfo`, `topology` |
| `pod` | `1` | reserved |
| `pod_net` | `2` | `/proc/net/*` when container detection succeeds |
| `container` | `3` | reserved |
| `unknown` | `4` | reserved |

`/proc/net/*` is network-namespace scoped. In a pod, these files describe the
pod network namespace, so network rows use `pod_net`. Disk and mount rows stay
`host` because they describe node block devices.

## Registry Sections

| `type_id` | Name | Source | Semantics | Sort key |
|-----------|------|--------|-----------|----------|
| `1_108_001` | `os_diskstats` | `/proc/diskstats` | `snapshot_full` | `(major, minor, ts)` |
| `1_109_001` | `os_netdev` | `/proc/net/dev` | `snapshot_full` | `(iface, ts)` |
| `1_110_001` | `os_snmp` | `/proc/net/snmp` | `snapshot_full` | `(ts)` |
| `1_111_001` | `os_netstat` | `/proc/net/netstat` | `snapshot_full` | `(ts)` |
| `1_112_001` | `os_mountinfo` | `/proc/self/mountinfo` + `statvfs(2)` | `on_change` | `(major, minor, mount_point, ts)` |
| `1_113_001` | `os_topology` | `/proc/cpuinfo` + `/sys/devices/system/cpu` | `on_change` | `(cpu_id, ts)` |

`1_112_001` and `1_113_001` use `on_change` with per-segment re-arm through
`SEGMENT_OPEN_SOURCES`, so each segment has the latest materialized rows.
Mountinfo may also be parsed during `OsCore` ticks for disk attribution; that
read path does not emit `1_112_001` rows.

## Collector Integration

Wave 2 extends the synchronous OS read path:

- `OsCore` collects counters: Wave 1 procfs sections, `diskstats`, `netdev`,
  `snmp`, and `netstat`.
- `OsMountTopo` emits `mountinfo` and `topology` as `on_change` sections.
- `mountinfo` is parsed whenever disk attribution needs the current device set.
  Emitted `os_mountinfo` rows keep the full mount table instead of collapsing
  devices to one display path.
- String dictionary IDs are resolved while OS rows are built, using the
  segment interner for device, interface, mount point, filesystem type, source,
  and model strings.

Disk cardinality is bounded by `KRONIKA_OS_MAX_DISKS` (default `256`). If the
cap is exceeded, the collector keeps the lowest `(major, minor)` devices and
logs `collection_degraded` with `reason=disk_cap`.

## Disk Attribution

In a container, `/proc/diskstats` reports node block devices. The collector
filters disk rows to devices that have at least one non-infrastructure mount in
`/proc/self/mountinfo`. Kubernetes infrastructure bind mounts such as
`/etc/hosts`, `/etc/hostname`, `/etc/resolv.conf`, `/dev/termination-log`,
`/run/secrets/`, and `/var/run/secrets/` do not make a device attributable to
the pod.

Mount entries with `major=0` are preserved during parsing. When the mount source
is a direct `/dev/<block>` node, the collector resolves the real device number
through `/sys/class/block/<device>/dev` using `KRONIKA_SYS_ROOT` when set.

`os_mountinfo` also records `total_bytes` and `free_bytes` from `statvfs(2)`.
Both fields are nullable because mounts can disappear or reject the syscall.
Mountinfo octal escapes are decoded before strings are interned and before
`statvfs(2)` is called.

## Parser Contracts

### `/proc/diskstats`

The parser accepts both modern 20-field rows and legacy 14-field rows.

Mandatory fields:

```text
major, minor, device,
reads, r_merged, read_sectors, read_time_ms,
writes, w_merged, write_sectors, write_time_ms,
io_in_progress, io_time_ms, io_weighted_time_ms
```

Optional fields:

```text
discards, d_merged, discard_sectors, discard_time_ms,
flushes, flush_time_ms
```

Rows with fewer than 14 fields are skipped. Non-numeric mandatory or present
optional fields return `ParseError`.

### `/proc/net/dev`

The parser keeps every interface, including `lo` and virtual interfaces, and
stores all 16 receive/transmit counters. It strips the interface colon and
handles rows where the first counter is glued to the interface token
(`eth0:1000 ...`).

### `/proc/net/snmp`

The parser reads `Tcp:` and `Udp:` two-line groups by column name, not by
position. Missing groups or missing keys default to `0`.

Stored TCP fields:

```text
ActiveOpens, PassiveOpens, AttemptFails, EstabResets, CurrEstab,
InSegs, OutSegs, RetransSegs, InErrs, OutRsts
```

Stored UDP fields:

```text
InDatagrams, OutDatagrams, InErrors, NoPorts
```

### `/proc/net/netstat`

The parser reads the `TcpExt:` two-line group by column name and ignores other
groups.

Stored fields:

```text
ListenOverflows, ListenDrops, TCPTimeouts, TCPFastRetrans,
TCPSlowStartRetrans, TCPOFOQueue, TCPSynRetrans
```

### `/proc/cpuinfo`

The parser emits one row per logical CPU block. It maps:

```text
processor -> cpu_id
model name -> model_name
core id -> core_id
physical id -> socket_id
```

The collector fills `mhz_max` from
`/sys/devices/system/cpu/cpu*/cpufreq/cpuinfo_max_freq`, converting kHz to MHz.
The field is `NULL` when sysfs does not expose a max-frequency value. `cpu MHz`
from `/proc/cpuinfo` is ignored because it can report the current scaled clock.

Missing numeric topology fields use sentinels: `core_id=-1`, `socket_id=-1`,
and `mhz_max=NULL`.

## BDD Coverage

The Wave 2 feature file uses fixture `/proc` and `/sys` trees, plus a statvfs
fixture, so assertions do not depend on the host running the tests. It covers:

- diskstats parsing and legacy rows with missing discard/flush counters;
- netdev, snmp, and netstat exact counter extraction;
- `scope=pod_net` for network sections in a container fixture;
- diskstats filtering by mountinfo in a pod fixture;
- btrfs-style `major=0` mount resolution through sysfs;
- full `os_mountinfo` mount-table emission with decoded mount paths;
- filesystem capacity fields on `os_mountinfo`;
- sysfs-backed `mhz_max` on `os_topology`.
