# OS Wave 3: process and cgroup metrics

This document records the Wave 3 OS metric contract and implementation shape.
It is a technical reference for the branch, not an execution checklist.

## Goal

Add process and cgroup visibility on top of Wave 1 and Wave 2:

- hot per-process rows from `/proc/PID/stat`, `/proc/PID/io`,
  `/proc/PID/schedstat`, `/proc/PID/comm`, `/proc/PID/cmdline`, and
  `/proc/PID/status`;
- less frequent `/proc/PID/status` rows;
- PID to cgroup mapping from `/proc/PID/cgroup`;
- cgroup v2 and v1 CPU, memory, I/O, and pids rows from `/sys/fs/cgroup`.

The collector keeps kernel-native units: scheduler ticks, nanoseconds,
microseconds, bytes, and raw counter values. Readers perform rates and display
conversion.

## Registry Sections

| `type_id` | Name | Source | Semantics | Sort key |
|-----------|------|--------|-----------|----------|
| `1_100_001` | `os_process` | `/proc/PID/*` hot set | `snapshot_full` | `(pid, starttime, ts)` |
| `1_101_001` | `os_process_status` | `/proc/PID/status` | `snapshot_full` | `(pid, starttime, ts)` |
| `1_200_001` | `os_cgroup_mapping` | `/proc/PID/cgroup` | `snapshot_full` | `(pid, starttime, ts)` |
| `1_201_001` | `os_cgroup_cpu` | cgroup cpu controller | `snapshot_full` | `(cgroup_path, ts)` |
| `1_202_001` | `os_cgroup_memory` | cgroup memory controller | `snapshot_full` | `(cgroup_path, ts)` |
| `1_203_001` | `os_cgroup_io` | cgroup io/blkio controller | `snapshot_full` | `(cgroup_path, major, minor, ts)` |
| `1_204_001` | `os_cgroup_pids` | cgroup pids controller | `snapshot_full` | `(cgroup_path, ts)` |

Every row carries `scope`: `host` when the collector runs on the host, and
`container` when container detection succeeds. Wave 3 does not claim pod-level
truth for process rows because `/proc` may be a host or container PID namespace
depending on the deployment.

## Process Contract

Process identity is `(pid, starttime)`. `starttime` is computed from
`/proc/stat` `btime`, the process start ticks, and `sysconf(_SC_CLK_TCK)`.
This prevents PID reuse from merging distinct processes.

`/proc/PID/io` is optional. The collector first reads it directly. On Linux, a
`PermissionDenied` read retries with `setfsuid` and `setfsgid` set to the
process real uid/gid, which grants access to foreign-UID processes
without requiring `CAP_SYS_PTRACE`. If the file is still unavailable, all I/O
columns in `1_100_001` are `NULL`.

`cmdline` is collected by default with NUL separators converted to spaces.
`environ` is not collected. If `cmdline` is empty or unavailable, only that
column is `NULL`; the process row is still emitted when required files parse.

Rows are capped before interning and encoding by `KRONIKA_OS_MAX_PROCS`
(default `4096`). The collector keeps the lowest numeric PIDs and logs
`collection_degraded` with `reason=process_cap` when rows are dropped.

## Cgroup Contract

The collector detects cgroup v2 when root v2 files exist under
`KRONIKA_SYS_ROOT/fs/cgroup`; otherwise it uses cgroup v1 controller trees.

cgroup v2 files:

- `cpu.stat`, `cpu.max`;
- `memory.current`, `memory.max`, `memory.stat`, `memory.events`;
- `io.stat`;
- `pids.current`, `pids.max`.

cgroup v1 normalization:

- `cpuacct.usage` nanoseconds to microseconds;
- `cpuacct.stat` ticks to microseconds through `sysconf(_SC_CLK_TCK)`;
- `cpu.stat throttled_time` nanoseconds to microseconds;
- `memory.stat` rss/cache/slab/kernel_stack fields to anon/file/slab/kernel;
- `memory.failcnt` to `max_events`;
- blkio service byte/op files to `rbytes`, `wbytes`, `rios`, `wios`.

cgroup walking is capped by `KRONIKA_OS_MAX_CGROUPS` (default `1024`),
`KRONIKA_OS_CGROUP_MAX_DEPTH` (default `8`), and
`KRONIKA_OS_MAX_CGROUP_IO_ROWS` (default `4096`). Cap hits log
`collection_degraded`.

## Collector Integration

Wave 3 adds independent source groups:

- `OsProcesses`, default interval 5 seconds;
- `OsProcessStatus`, default interval 30 seconds;
- `OsCgroup`, default interval 10 seconds;
- `OsCgroupMapping`, default interval 30 seconds.

The first tick and SIGUSR2 forced ticks read every source, so a fresh segment
can contain all process and cgroup sections. String dictionary IDs are resolved
while rows are built, after caps have been applied.

- Process fd count, `wchan`, `oom_score`, maps/smaps, cpuset, and hugetlb.
- A wall-clock budget for procfs/sysfs traversal in addition to row caps.
