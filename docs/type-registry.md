# Реестр `type_id`

Версия документа: draft-4, 2026-06-12.

Этот набор документов — единственное место, где выдаются `type_id` для
формата PGM. Он дополняет [формат сегмента](segment-format.md): формат описывает
контейнер, а реестр описывает типизированные секции внутри контейнера.

Контракт типа включает:

- числовой `type_id`;
- байтовую раскладку секции;
- семантику сбора;
- ключ сортировки;
- классификацию колонок для механизма сравнения и построения графиков.

`type_id` не переиспользуются. Удалённые или больше не пишущиеся типы остаются
в реестре с пометкой `deprecated`.

## Быстрая карта диапазонов

| Диапазон      | Содержимое |
|---------------|------------|
| `1_001_001+`  | PostgreSQL-снимки |
| `1_100_001+`  | ОС: `/proc`, `/sys`, сеть, диски |
| `1_200_001+`  | cgroup |
| `2_001_001+`  | секция событий |
| `3_001_001+`  | словари |
| `10_001_001+` | графики |

Полная схема нумерации описана в
[правилах и семантике](type-registry/semantics.md).
