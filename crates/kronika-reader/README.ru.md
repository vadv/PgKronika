# kronika-reader

[English version](README.md)

`kronika-reader` — ядро чтения PGM-сегментов. `PgmUnit` декодирует
PGM-контейнер через любой `ReadAt`-источник — sealed-файл или in-memory-часть
журнала — одним путём. `LocalDirSnapshot` объединяет sealed-сегменты с
live-частями `active.parts`, исключает точные sealed/live-дубликаты и отдаёт
диагностику скана.

## Открытие сегмента

`Segment::open(path)` читает последние 8 байт, получает длину каталога, читает
блок перед ними и проверяет CRC. Если `catalog_len` больше лимита или выходит за
границы файла, сегмент отвергается до выделения памяти. `Segment::catalog()`
возвращает записи каталога; тела секций читаются по требованию.

## Декодирование секции

`Segment::decode(entry)` читает тело по `entry.offset` (ограничено
`MAX_SECTION_BYTES`), сверяет его с `entry.crc32c` и декодирует через
`kronika_registry::decode_any`.

CRC проверяется до передачи байтов в Parquet-парсер. Код чтения передаёт
`kronika_format::crc32c` в `VerifiedSection::verify`; при этом
`kronika-registry` не зависит от `kronika-format`.

## Чтение строк

Колонки снимков хранят `str_id`, а не байты строки. `Segment::dictionary()`
читает `dict.strings` и `dict.blobs` в карту `str_id -> значение`. Словарные
секции проходят проверку CRC и те же лимиты строк и групп строк, что и секции
данных.

`Dictionary::resolve(str_id)` возвращает полную строку или blob-значение с
`full_len` и `truncated`. Так вызывающий код не примет сохранённый префикс за
исходное значение целиком. Если один id встретился как строка и как blob в
разных частях, используется blob: в нём есть метаданные усечения.

## Снимки локального каталога

`LocalDirSnapshot::units()` возвращает сначала sealed-сегменты, затем live-части
журнала. Live-часть скрывается только при точном совпадении её каталога с
каталогом sealed-сегмента. Пересечение временных диапазонов не доказывает, что
live-часть была запечатана.

`warnings()` возвращает предупреждения о пропущенных файлах и частях.
`damages()` возвращает типизированные `DamageRegion` для повреждённых диапазонов
`active.parts`; валидные части вокруг повреждения остаются видимыми.

## Границы

Этот крейт не предоставляет запросы по времени, детализацию, межсегментный кэш
`str_id` и чтение `dict.blobs` по требованию. Эти API находятся уровнем выше
`Segment`, `PgmUnit` и `LocalDirSnapshot`.
