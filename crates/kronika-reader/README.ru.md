# kronika-reader

[English version](README.md)

`kronika-reader` открывает запечатанный PGM-сегмент и декодирует его секции. Он
читает концевой каталог из хвоста файла, затем читает тело каждой секции по её
диапазону из каталога — позиционно и ограниченно, никогда не загружая файл
целиком (segment-format.md, «Чтение из S3»).

## Открытие сегмента

`Segment::open(path)` читает последние 8 байт (хвостовой указатель) ради длины
каталога, читает блок каталога перед ним и проверяет CRC каталога. Каталог —
это хвост сегмента, и он ограничен: `catalog_len` больше лимита или вне файла —
это повреждённый хвост, отклоняется до любой аллокации. `Segment::catalog()`
возвращает декодированные записи; тела секций читаются по требованию.

## Декодирование секции

`Segment::decode(entry)` читает тело по `entry.offset` (ограничено
`MAX_SECTION_BYTES`), сверяет его с `entry.crc32c` и декодирует через
`kronika_registry::decode_any`.

Это боевое применение CRC-границы доверия реестра: ридер вставляет
`kronika_format::crc32c` в `VerifiedSection::verify`, поэтому подделанное или
испорченное тело отвергается до того, как его увидит Parquet-парсер, а реестр не
зависит от крейта формата (разделение по варианту C).

## Что ещё не реализовано

Резолвинг строк (чтение `dict.strings` / `dict.blobs` для отображения `str_id` в
байты), запросы по времени и детализации, межсегментный кэш `str_id` и путь
чтения диапазонами из S3 появятся позже.
