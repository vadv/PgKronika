# kronika-writer

[English version](README.md)

`kronika-writer` отвечает за путь записи коллектора: принимает данные от
источников и доводит их до запечатанного `.pgm`-сегмента. Байтовые структуры
описывает `kronika-format`; этот крейт хранит состояние, которое нужно только
во время записи.

## Что уже есть

Крейт сейчас экспортирует:

- `Interner` — интернер строк одного сегмента поверх
  `kronika_format::SegmentDicts`.

Интернер использует модель словарей из `kronika-format` и добавляет поведение,
которое нужно только коду записи:

- `take_new()` возвращает идентификаторы, впервые интернированные с прошлого
  вызова. Они нужны для словаря следующей мини-части: мини-часть хранит строки,
  которые впервые встретились после предыдущего сброса.
- `seal()` передаёт словари на запечатывание и начинает следующий сегмент с
  пустого интернера. Поэтому коллектор может досрочно запечатать сегмент, если
  интернер быстро растёт.
- `stats()` возвращает размеры словарей для собственных метрик коллектора.

Если интернирование завершилось ошибкой — коллизией или конфликтом размещения,
— словари и список новых идентификаторов не меняются.

## Чего здесь пока нет

Буферы по типам, журнал `active.parts`, слияние, запечатывание и
Parquet-кодирование появятся в следующих шагах. Правила словарей —
выбор словаря, усечение и обработка коллизий — описаны в `kronika-format`.
